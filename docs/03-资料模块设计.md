# 资料模块设计

## 1. 模块目标

资料模块负责课程资料的本地存储、元数据管理、文件树展示和内容访问，目标如下：

1. 为每门课程分配独立资料空间。
2. 支持文件夹树结构管理。
3. 支持文件上传到本地。
4. 支持文件元信息查看、下载和预览。
5. 支持基于课程角色控制资料访问。
6. 为 Agent 模块提供可检索资料来源。

## 2. 模块边界

资料模块负责：

1. 资料空间管理。
2. 文件与目录元数据管理。
3. 本地文件路径映射。
4. 预览、下载、删除。

资料模块不负责：

1. 用户登录。
2. 课程成员关系维护。
3. Agent 回答生成。

## 3. 数据库表结构

### 3.1 课程资料空间表 `course_storage_spaces`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| course_id | BIGINT | 课程 ID，唯一 |
| root_path | VARCHAR(255) | 本地根目录 |
| quota_bytes | BIGINT | 空间配额 |
| used_bytes | BIGINT | 已使用大小 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 3.2 资料节点表 `course_material_nodes`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| course_id | BIGINT | 课程 ID |
| space_id | BIGINT | 资料空间 ID |
| parent_id | BIGINT | 父节点 ID，根节点为空 |
| node_type | VARCHAR(16) | `folder` 或 `file` |
| node_name | VARCHAR(255) | 节点名称 |
| file_ext | VARCHAR(32) | 文件扩展名 |
| storage_path | VARCHAR(512) | 本地存储路径 |
| mime_type | VARCHAR(128) | 文件类型 |
| file_size | BIGINT | 文件大小 |
| latest_version_no | INT | 最新版本号 |
| sort_index | INT | 排序值 |
| is_deleted | TINYINT | 是否删除 |
| created_by | BIGINT | 创建人 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 3.3 资料版本表 `course_material_versions`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| material_node_id | BIGINT | 文件节点 ID |
| version_no | INT | 版本号 |
| storage_path | VARCHAR(512) | 当前版本文件路径 |
| file_size | BIGINT | 文件大小 |
| mime_type | VARCHAR(128) | 文件类型 |
| upload_user_id | BIGINT | 上传人 |
| created_at | DATETIME | 上传时间 |

## 4. 参考 SQL

```sql
CREATE TABLE course_storage_spaces (
    id BIGINT PRIMARY KEY,
    course_id BIGINT NOT NULL UNIQUE,
    root_path VARCHAR(255) NOT NULL,
    quota_bytes BIGINT NOT NULL DEFAULT 0,
    used_bytes BIGINT NOT NULL DEFAULT 0,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    FOREIGN KEY (course_id) REFERENCES courses(id)
);

CREATE TABLE course_material_nodes (
    id BIGINT PRIMARY KEY,
    course_id BIGINT NOT NULL,
    space_id BIGINT NOT NULL,
    parent_id BIGINT,
    node_type VARCHAR(16) NOT NULL,
    node_name VARCHAR(255) NOT NULL,
    file_ext VARCHAR(32),
    storage_path VARCHAR(512),
    mime_type VARCHAR(128),
    file_size BIGINT NOT NULL DEFAULT 0,
    latest_version_no INT NOT NULL DEFAULT 1,
    sort_index INT NOT NULL DEFAULT 0,
    is_deleted TINYINT NOT NULL DEFAULT 0,
    created_by BIGINT NOT NULL,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    FOREIGN KEY (course_id) REFERENCES courses(id),
    FOREIGN KEY (space_id) REFERENCES course_storage_spaces(id),
    FOREIGN KEY (parent_id) REFERENCES course_material_nodes(id),
    FOREIGN KEY (created_by) REFERENCES users(id)
);

CREATE TABLE course_material_versions (
    id BIGINT PRIMARY KEY,
    material_node_id BIGINT NOT NULL,
    version_no INT NOT NULL,
    storage_path VARCHAR(512) NOT NULL,
    file_size BIGINT NOT NULL,
    mime_type VARCHAR(128),
    upload_user_id BIGINT NOT NULL,
    created_at DATETIME NOT NULL,
    UNIQUE (material_node_id, version_no),
    FOREIGN KEY (material_node_id) REFERENCES course_material_nodes(id),
    FOREIGN KEY (upload_user_id) REFERENCES users(id)
);
```

## 5. 功能说明

### 5.1 课程资料空间初始化

每门课程创建后自动分配独立资料空间。

例如：

1. 课程 A 对应目录 `/storage/course/1001/`
2. 课程 B 对应目录 `/storage/course/1002/`

### 5.2 文件树管理

系统通过资料节点表维护树结构，前端可直接请求树形数据进行展示。

支持节点类型：

1. 文件夹
2. 文件

### 5.3 文件上传

教师和创建者上传文件到课程空间，系统完成：

1. 保存本地文件。
2. 写入资料节点。
3. 写入版本记录。
4. 更新空间已用容量。

### 5.4 文件预览

学生、教师、创建者可在权限范围内预览文件内容。

预览能力可按文件类型区分：

1. 文本：直接读取展示。
2. PDF：页级预览。
3. 图片：直接展示。
4. Office 文档：可转预览格式后展示，或只提供下载。

### 5.5 文件下载

课程成员可下载有权限访问的资料。

### 5.6 删除资料

教师和创建者可删除资料。

建议逻辑删除，保留历史记录用于 Agent 引用追踪和审计。

## 6. 接口设计

本模块所有接口响应都应遵循统一返回格式：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 6.1 获取文件树

接口：`GET /api/courses/{courseId}/materials/tree`

功能：

1. 返回课程资料目录树。

响应内容建议：

```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": 1,
      "name": "第1章",
      "type": "folder",
      "children": [
        {
          "id": 2,
          "name": "课件.pdf",
          "type": "file",
          "fileSize": 102400
        }
      ]
    }
  ]
}
```

实现要点：

1. 只返回未删除节点。
2. 校验当前用户是否为课程成员。
3. 建议按父子关系构建树，避免直接扫描磁盘目录。

接口范围：

1. 只返回结构与基础元信息。
2. 不直接返回文件内容。

权限规则：

1. 课程成员可调用。

### 6.2 创建文件夹

接口：`POST /api/courses/{courseId}/materials/folders`

功能：

1. 在指定目录下创建文件夹节点。

请求参数建议：

```json
{
  "parentId": 1,
  "folderName": "作业参考资料"
}
```

实现要点：

1. 校验父节点存在且属于当前课程。
2. 同级目录下名称建议唯一。

接口范围：

1. 只创建目录节点。

权限规则：

1. 教师、创建者可调用。

### 6.3 上传文件

接口：`POST /api/courses/{courseId}/materials/upload`

功能：

1. 向课程资料空间上传文件。

实现要点：

1. 校验上传者课程角色。
2. 校验课程空间剩余容量。
3. 保存文件到本地路径。
4. 建立文件节点与版本记录。

接口范围：

1. 一次上传一个或多个文件均可，但建议先按单文件设计，便于失败处理。

权限规则：

1. 教师、创建者可调用。

### 6.4 获取资料详情

接口：`GET /api/courses/{courseId}/materials/{nodeId}`

功能：

1. 查看文件或目录元信息。

实现要点：

1. 校验节点属于当前课程。
2. 返回创建人、更新时间、文件大小、类型等信息。

接口范围：

1. 只返回元数据。

权限规则：

1. 课程成员可调用。

### 6.5 预览资料

接口：`GET /api/courses/{courseId}/materials/{nodeId}/preview`

功能：

1. 获取预览内容或预览地址。

实现要点：

1. 校验课程成员身份。
2. 根据文件类型分流处理。
3. 只允许预览未删除文件。

接口范围：

1. 仅服务于展示，不等同于下载原文件。

权限规则：

1. 课程成员可调用。

### 6.6 下载资料

接口：`GET /api/courses/{courseId}/materials/{nodeId}/download`

功能：

1. 下载指定文件。

实现要点：

1. 校验节点类型必须是文件。
2. 校验权限和文件存在性。

接口范围：

1. 仅适用于文件节点。

权限规则：

1. 课程成员可调用。

### 6.7 重命名或更新资料信息

接口：`PUT /api/courses/{courseId}/materials/{nodeId}`

功能：

1. 修改文件或文件夹名称。
2. 可扩展修改排序值、Agent 可检索标记等。

实现要点：

1. 不允许越权修改其他课程节点。
2. 保证同级目录下命名规则。

接口范围：

1. 仅修改元数据，不修改文件内容。

权限规则：

1. 教师、创建者可调用。

### 6.8 删除资料

接口：`DELETE /api/courses/{courseId}/materials/{nodeId}`

功能：

1. 删除文件或文件夹。

实现要点：

1. 建议逻辑删除。
2. 删除文件夹时要联动处理子节点。
3. 若资料已被 Agent 引用，仍建议保留历史版本信息。

接口范围：

1. 仅删除课程内资料节点。

权限规则：

1. 教师、创建者可调用。

## 7. 权限规则

1. 学生：可查看树、详情、预览、下载。
2. 教师：拥有学生全部权限，并可创建目录、上传、修改、删除资料。
3. 创建者：拥有教师全部权限。

## 8. 实现注意事项

1. 逻辑目录树应与真实物理路径解耦，便于后续迁移存储方式。
2. 文件权限判断必须先查课程成员关系，再查资料节点。
3. 资料预览接口应考虑大文件限流和访问审计。
4. 如果 Agent 需要更精细资料范围，可在资料表增加 `agent_enabled` 字段。
