# 学习计划模块设计

## 1. 模块目标

学习计划模块负责为课程成员生成课程内的个人学习计划，目标如下：

1. 用户可输入学习目标和每日可用时间。
2. 系统基于课程资料和用户输入，让模型自行决定合理天数并生成按天拆分的学习计划。
3. 一个课程内，一个学生可以拥有多个学习计划。
4. 一个学习计划可以拥有多个步骤。
5. 学习计划默认只对当前用户可见。
6. 用户可标记步骤完成状态。

## 2. 模块边界

学习计划模块负责：

1. 课程内个人学习计划生成。
2. 每日学习任务拆分。
3. 计划列表、计划详情与任务完成状态维护。

学习计划模块不负责：

1. 课程资料上传与编辑。
2. 用户登录与登录态管理。
3. Agent 对话记录管理。

## 3. 数据库表结构

### 3.1 学习计划表 `study_plans`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| course_id | BIGINT | 课程 ID |
| user_id | BIGINT | 计划所属用户 |
| goal | TEXT | 学习目标 |
| deadline_date | DATE | 计划结束日期，由模型生成的最后一步日期派生，当前不由用户输入 |
| daily_minutes | INT | 每日可用时间（分钟） |
| status | VARCHAR(16) | 当前实现默认保存为 `active` |
| generated_summary | TEXT | 计划摘要 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 3.2 学习计划任务表 `study_plan_items`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| plan_id | BIGINT | 学习计划 ID |
| day_index | INT | 第几天 |
| plan_date | DATE | 计划日期 |
| title | VARCHAR(255) | 当天任务标题 |
| tasks_text | TEXT | 当天任务内容 |
| suggested_minutes | INT | 建议学习时长 |
| material_node_ids | TEXT | 关联资料节点 ID 列表（JSON） |
| status | VARCHAR(16) | `pending` / `done` |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

## 4. 功能说明

### 4.1 生成学习计划

用户输入：

1. 学习目标
2. 每日可用时间

系统处理流程：

1. 校验用户是否为课程成员。
2. 读取课程资料摘要。
3. 调用学习计划生成器生成结构化计划，由模型自行判断计划天数。
4. 若模型不可用或返回格式不合法，则返回生成服务不可用。
5. 保存新的计划和步骤；历史计划不会被归档或覆盖。

### 4.2 查看计划列表

用户进入课程计划页后，可查看当前用户在该课程内创建的学习计划列表。

列表展示：

1. 学习目标
2. 计划天数
3. 每日可用时间
4. 步骤完成进度

### 4.3 查看计划详情

用户点击计划后，可查看计划摘要以及步骤列表。

### 4.4 更新任务状态

用户可将单日任务标记为完成或恢复为未完成。

## 5. 接口设计

### 5.1 获取学习计划列表

接口：`GET /api/courses/{courseId}/study-plans`

功能：

1. 获取当前用户在课程内的学习计划列表。
2. 返回每个计划的步骤总数与已完成步骤数。

### 5.2 获取学习计划详情

接口：`GET /api/courses/{courseId}/study-plans/{planId}`

功能：

1. 获取指定计划的基础信息。
2. 获取该计划下的全部步骤。

### 5.3 生成学习计划

接口：`POST /api/courses/{courseId}/study-plans/generate`

请求示例：

```json
{
  "goal": "两周内掌握第一到第三章",
  "dailyMinutes": 60
}
```

### 5.4 更新单日任务状态

接口：`PATCH /api/courses/{courseId}/study-plans/{planId}/items/{itemId}`

请求示例：

```json
{
  "status": "done"
}
```

## 6. 权限规则

1. 课程成员可生成自己的学习计划。
2. 课程成员只能查看和修改自己的学习计划与步骤。
3. 非课程成员不可访问。
