# 课程模块设计

## 1. 模块目标

课程模块负责管理课程实体及课程内成员角色关系，目标如下：

1. 支持课程创建。
2. 支持课程列表与详情查看。
3. 支持课程信息编辑。
4. 支持课程成员管理。
5. 支持按课程角色实施权限控制。
6. 支持课程删除。

## 2. 模块边界

课程模块负责：

1. 课程基础信息维护。
2. 课程成员与角色维护。
3. 课程级权限判定基础。

课程模块不负责：

1. 用户注册登录。
2. 文件真实上传与预览处理。
3. Agent 问答生成。

## 3. 数据库表结构

### 3.1 课程表 `courses`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| course_code | VARCHAR(64) | 课程编号，唯一 |
| course_name | VARCHAR(128) | 课程名称 |
| course_description | TEXT | 课程说明 |
| owner_user_id | BIGINT | 创建者用户 ID |
| status | VARCHAR(16) | 课程状态，`active`/`archived`/`deleted` |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 3.2 课程成员表 `course_members`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT | 主键 |
| course_id | BIGINT | 课程 ID |
| user_id | BIGINT | 用户 ID |
| role | VARCHAR(16) | 角色，`owner`/`teacher`/`student` |
| join_status | VARCHAR(16) | 成员状态，`active`/`removed` |
| joined_at | DATETIME | 加入时间 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

## 4. 参考 SQL

```sql
CREATE TABLE courses (
    id BIGINT PRIMARY KEY,
    course_code VARCHAR(64) NOT NULL UNIQUE,
    course_name VARCHAR(128) NOT NULL,
    course_description TEXT,
    owner_user_id BIGINT NOT NULL,
    status VARCHAR(16) NOT NULL DEFAULT 'active',
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    FOREIGN KEY (owner_user_id) REFERENCES users(id)
);

CREATE TABLE course_members (
    id BIGINT PRIMARY KEY,
    course_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    role VARCHAR(16) NOT NULL,
    join_status VARCHAR(16) NOT NULL DEFAULT 'active',
    joined_at DATETIME NOT NULL,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    UNIQUE (course_id, user_id),
    FOREIGN KEY (course_id) REFERENCES courses(id),
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## 5. 功能说明

### 5.1 创建课程

用户创建课程后：

1. 生成课程记录。
2. 自动写入课程成员表，角色为 `owner`。
3. 后续可触发资料空间初始化和 Agent 初始化。

### 5.2 查看课程列表

用户查看自己加入或创建的课程。

### 5.3 查看课程详情

课程成员可查看课程基础信息。

### 5.4 编辑课程信息

教师和创建者可修改课程名称、介绍等字段。

### 5.5 管理课程成员

教师和创建者可添加学生、设置教师、移除成员。

### 5.6 删除课程

仅创建者可删除课程。

建议采用逻辑删除：

1. 课程状态置为 `deleted`。
2. 成员、资料、Agent 数据保留用于审计。

## 6. 接口设计

### 6.1 创建课程

接口：`POST /api/courses`

功能：

1. 创建一门课程。

请求参数建议：

```json
{
  "courseCode": "CS101",
  "courseName": "计算机导论",
  "courseDescription": "课程简介"
}
```

实现要点：

1. 校验课程编号唯一。
2. 当前用户成为创建者。
3. 同步创建课程成员关系。
4. 可在事务中创建资料空间和课程 Agent 初始配置。

接口范围：

1. 仅创建课程本身。
2. 不在该接口中处理批量导入学生。

权限规则：

1. 仅登录用户可调用。

### 6.2 获取课程列表

接口：`GET /api/courses`

功能：

1. 返回当前用户参与的课程列表。

实现要点：

1. 基于课程成员表查询。
2. 可返回当前用户在课程中的角色。

接口范围：

1. 仅返回当前用户可见课程。

权限规则：

1. 仅登录用户可调用。

### 6.3 获取课程详情

接口：`GET /api/courses/{courseId}`

功能：

1. 查看课程详情。

实现要点：

1. 先校验是否为课程成员。
2. 返回课程基础信息与当前用户角色。

接口范围：

1. 只返回单门课程详情。

权限规则：

1. 课程成员可调用。

### 6.4 编辑课程

接口：`PUT /api/courses/{courseId}`

功能：

1. 修改课程名称、描述等信息。

请求参数建议：

```json
{
  "courseName": "计算机导论（2026春）",
  "courseDescription": "更新后的课程说明"
}
```

实现要点：

1. 校验当前用户是否为 `teacher` 或 `owner`。
2. 限制可修改字段范围，避免误改创建者等关键关系字段。

接口范围：

1. 仅修改课程基本信息。

权限规则：

1. 教师、创建者可调用。

### 6.5 删除课程

接口：`DELETE /api/courses/{courseId}`

功能：

1. 删除课程。

实现要点：

1. 仅允许创建者删除。
2. 建议逻辑删除。
3. 删除后课程不可再对学生可见。

接口范围：

1. 只删除课程业务实体。
2. 是否同步冻结资料和 Agent，由业务规则决定，建议同步冻结。

权限规则：

1. 仅创建者可调用。

### 6.6 查看成员列表

接口：`GET /api/courses/{courseId}/members`

功能：

1. 查看课程成员信息。

实现要点：

1. 教师、创建者可查看完整列表。
2. 学生可只查看教师信息和自己信息，避免泄露不必要的学生列表。

接口范围：

1. 仅返回成员及角色信息。

权限规则：

1. 课程成员可调用，但返回范围受角色约束。

### 6.7 添加成员

接口：`POST /api/courses/{courseId}/members`

功能：

1. 向课程添加学生或教师。

请求参数建议：

```json
{
  "userId": 1002,
  "role": "student"
}
```

实现要点：

1. 校验目标用户存在。
2. 校验该用户未在课程中重复加入。
3. 若已存在被移除记录，可恢复为有效成员。

接口范围：

1. 一次添加一个成员，便于错误处理。

权限规则：

1. 教师、创建者可调用。

### 6.8 修改成员角色

接口：`PUT /api/courses/{courseId}/members/{memberId}`

功能：

1. 调整课程成员角色。

实现要点：

1. 不允许把创建者直接降级为普通学生，除非先完成所有权转移。
2. 教师是否可提升他人为教师，可按业务策略限定为仅创建者可操作。

接口范围：

1. 只修改成员角色和状态。

权限规则：

1. 建议仅创建者可修改教师/创建者级别角色。
2. 教师可移除学生或添加学生。

### 6.9 移除成员

接口：`DELETE /api/courses/{courseId}/members/{memberId}`

功能：

1. 将成员移出课程。

实现要点：

1. 创建者不可被普通教师移除。
2. 已移除成员失去课程、资料、Agent 访问权限。

接口范围：

1. 只处理成员关系，不删除用户。

权限规则：

1. 教师可移除学生。
2. 创建者可移除教师和学生。

## 7. 权限规则

1. 学生：只读课程信息。
2. 教师：可编辑课程、管理学生、管理资料与 Agent。
3. 创建者：拥有课程最高权限，包括删除课程和管理教师。

## 8. 实现注意事项

1. 课程权限建议统一封装为角色判断能力，供资料和 Agent 模块复用。
2. 删除课程后，相关接口应统一返回“课程不存在”或“课程已删除”。
3. 成员变更要即时生效，否则会影响资料访问和 Agent 访问控制。
